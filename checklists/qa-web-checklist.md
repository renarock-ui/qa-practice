# QA Web Application Checklist

## 1. Functional Testing
- [ ] All buttons are clickable and perform expected actions
- [ ] Forms submit correctly with valid data
- [ ] Required fields are properly validated
- [ ] Error messages are displayed when validation fails
- [ ] Links redirect to correct destinations

---

## 2. Input Validation
- [ ] Empty fields are handled correctly
- [ ] Invalid data formats are rejected
- [ ] Character limits are enforced
- [ ] Special characters are handled safely

---

## 3. UI / UX Validation
- [ ] Text is readable and properly aligned
- [ ] Buttons and inputs are clearly visible
- [ ] Error messages are clear and user-friendly
- [ ] Layout is consistent across pages

---

## 4. Cross-Browser / Device
- [ ] Application works correctly on Chrome
- [ ] Application works correctly on mobile view
- [ ] No layout breaks on different screen sizes

---

## 5. Performance (Basic)
- [ ] Pages load within acceptable time
- [ ] No unnecessary loading delays
- [ ] No broken images or missing assets

---

## 6. Regression Check
- [ ] Existing functionality still works after changes
- [ ] No new bugs introduced in related areas
